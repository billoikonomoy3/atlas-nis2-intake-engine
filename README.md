# Atlas — NIS2 Stage-3 Intake Engine

Automates everything a firm can do **before** issuing the PBC ("Provided-By-Client")
request list on a NIS2 readiness engagement, from user-provided facts alone:
applicability & classification → assessment scope → tailored criteria set →
proportionality → draft PBC list. Offline, deterministic, audit-grade. No LLM,
no network, no external libraries.

> Turns a generic regulation into a specific, finite, defensible checklist for *this* client.

## Why this exists (and what the v1 toy got wrong)

A naive scope checker uses the practitioner shorthand "≥50 staff **OR** ≥€10m
turnover". That produces **negligent verdicts**. This engine encodes the real
Stage-3 logic explicitly:

| Trap | Naive tool | Atlas |
|---|---|---|
| **Size test** | staff OR turnover | Staff is a hard ceiling (≥); the two financials are **one limb read in the entity's favour** — breached only by exceeding **BOTH** turnover AND balance sheet (>). 2003/361/EC. |
| **40 staff / €15m turnover / €5m balance** | wrongly *in scope* | correctly **small / out of scope** (balance only €5m) — the canonical false-positive trap, now a regression test |
| **Group structure** | ignored | **Article 6 recursive aggregation**: +100% of linked (controlled) enterprises, pro-rata of partners (25–50%), consolidated down the whole tree — a subsidiary of a multinational is never a microenterprise |
| **One anomalous year** | flips scope | **Article 4(2) two-year rule**: a band change only takes effect across two consecutive periods |
| **Article 3 carve-outs** | silently deferred | **encoded**: QTSP/DNS/TLD = essential at any size; medium/large e-comms = essential (not merely important); Art 2(2) designation = essential at any size (or held *pending* the act); central-/local-government bodies in scope regardless of size |

## What's in the box

| File | Role |
|---|---|
| `atlas_nis2_intake.html` | **The demoable artifact.** Single self-contained file — open in any browser, runs from a USB stick in a client room. Stepper UI, live verdict, group-tree builder, traced scope memo, draft PBC list, print-to-PDF + JSON snapshot export. Embeds the engine and runs a **32-case self-test on load** (green badge). |
| `scope.py` | The deterministic engine (the "gold layer"): recursive Article 6 aggregation, size band, two-year rule, classification + Article 3/2(2) overrides. Pure functions, full audit trail per verdict. |
| `criteria.py` | NIS2 Art 20 / 21(2)(a–j) / 23 → NIST CSF 2.0 criteria set (16 rows), 5-level maturity, evidence, PBC items; transparent proportionality model. |
| `test_scope.py` | 32-case test catalogue proving the engine correct (`python test_scope.py` → `32 passed`). |

## Two decisions that make it defensible (interview talking points)

1. **Correctness of the size logic is the whole game.** A wrong applicability
   verdict is a negligent verdict. The boolean logic, group aggregation, two-year
   rule, and Article 3 overrides are encoded explicitly and pinned by 32 tests —
   *and the verdict ships with its own audit trail*, so every conclusion is
   re-derivable, not asserted.

2. **ISO/IEC 27001 control text is copyrighted by ISO — so it is never reproduced.**
   The public-facing criteria are keyed only to (a) paraphrased **NIS2** prose (EU
   law) and (b) **NIST CSF 2.0** function/subcategory IDs (US Government, public
   domain). ISO is referenced by **clause number only** (e.g. `8.16`) as a
   *licensed internal crosswalk* — a factual pointer a licensed user can resolve,
   carrying no protected expression. Commercial awareness baked into the design.

## How it was built (and verified)

The content (logic spec + 32 test cases, the NIS2→NIST CSF crosswalk, the
proportionality model, the intake schema) was generated by a fan-out of
specialist agents, then **adversarially cross-checked**. That review earned its
keep — it caught:

- **(high)** the Article 6 aggregation had to be made **recursive**, or multi-tier
  groups under-count and a controlled subsidiary wrongly falls out of scope; ← fixed
- **(med)** test control/linked **before** partner routing (control can exist ≤50%);
- a NIST CSF **1.1→2.0 error** (`ID.BE-05` doesn't exist in CSF 2.0 → `GV.OC-04`); ← fixed
- ISO-text leakage scan: **clean**.

## Complex corporate scenarios it handles

Multinational subsidiary (linked → large), JV/partner pro-rata, multi-entity
multi-geography groups, sole-provider/systemic Art 2(2) designation, e-comms /
QTSP / DNS / TLD special types, single-anomalous-year, and below-medium-but-
overridden — each with the controlling basis isolated in the memo. Try the
**Load a worked scenario** menu on the cover screen.

## Run it

```bash
python test_scope.py     # prove the engine: 32 passed
python scope.py          # quick CLI demo (trap + subsidiary)
python criteria.py       # criteria coverage + a proportionality example
```
Then open **`atlas_nis2_intake.html`** in a browser for the full interactive demo.

## Honest limitations (flagged in-tool for human review)

First-pass triage, not legal advice. The "central-government ⇒ essential" class is
flagged as **transposition-dependent**; deep partner-of-partner chaining and
mixed-sector entities are flagged for manual review; sector lists are top-level
Annex groupings to verify against the official text. The tool defers what it
should defer — and says so on the face of the memo.
